How It Works

Three lines of code.
Complete time-travel.

1

Record

Add the probe to your Express app. Every HTTP request, DB query, random value, and timer is captured at I/O boundaries — with <1% CPU overhead.

→
2

Replay

Download the recording. Run it locally. All I/O is mocked with recorded values. The app executes exactly as it did in production — deterministically.

→
3

Fix

Step through the timeline. See every DB query result, every API response, every timestamp. Find the bug in 5 minutes — not 5 hours.

app.ts
import { ErgenekonProbe } from '@ergenekon/probe';

const probe = new ErgenekonProbe({
  serviceName: 'checkout-service',
});

app.use(probe.middleware()); // That's it. You're recording.

Architecture

One pipeline.
End-to-end determinism.

From production HTTP request to byte-perfect local replay —
five decoupled packages, zero shared state.

npm 🔬
PROBE
@ergenekon/probe
Express middleware — intercepts HTTP, DB, DNS, FS, timers, globals at I/O boundaries
server 📦
COLLECTOR
@ergenekon/collector
Ingest server — rate-limited, checksummed, durable writes to disk in NDJSON + binary
shared ⚛️
CORE
@ergenekon/core
HLC clock · ULID IDs · Ed25519 licenses · shared types — zero runtime deps
engine ⏪
REPLAY
@ergenekon/replay
Deterministic re-execution — mocks all I/O with recorded values, byte-for-byte identical
UI 🖥️
DASHBOARD
@ergenekon/ui
Time-travel debugger — session list, timeline scrubber, service flow, diff view

🔬 Probe intercepts

  • HTTP incoming / outgoing
  • Node:fs — read/write/stat
  • Node:dns — lookup/resolve
  • Database queries (pg, mysql2)
  • Date.now() · Math.random()
  • setTimeout / setInterval

📦 Collector guarantees

  • Token bucket: 100 req/min/IP
  • SHA-256 checksum per session
  • Durable write: fsync before ACK
  • NDJSON + binary .prdx format
  • HLC timestamp ordering
  • Spill buffer on disconnect

⚛️ Core guarantees

  • Hybrid Logical Clock (HLC)
  • ULID — monotonic, sortable IDs
  • Ed25519 license signing
  • Zero external dependencies
  • Strict TypeScript types
  • 213 tests, 0 flaky

⏪ Replay guarantees

  • Byte-for-byte identical output
  • All I/O mocked from recording
  • Deep diff on response body
  • Multi-service trace alignment
  • Property-based tested
  • No network in replay mode
